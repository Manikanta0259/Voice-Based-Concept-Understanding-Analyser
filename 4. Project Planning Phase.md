# 4. Project Planning Phase

## Purpose
Plan, milestones, tasks and timeline for delivering VBCUA features.

## Suggested milestones (example)
- Sprint 1 (Week 1): Stabilize audio ingestion & transcription (audio_utils + speech_to_text). Add unit tests for audio loader.
- Sprint 2 (Week 2): Implement semantic similarity and keyword coverage; basic scoring function integration; UI wireframe for analysis flow.
- Sprint 3 (Week 3): PDF report layout, saving results, and user history.
- Sprint 4 (Week 4): Polish UI, add sample data, error handling, and basic deployment instructions.

## Sample tasks
- Write unit tests for analyze_audio_features and load_and_preprocess_audio.
- Add a synthetic audio sample set (extend `source/generate_samples.py`) for CI tests.
- Harden DB migrations and add export/import scripts for backup.
- Add environment variable handling and a .env template for API keys (Whisper).

## Risk register & mitigations
- ASR model size & speed: Allow selection of smaller models (tiny, base) for demo; provide guidance in README.
- Missing ffmpeg binary: Add clear instructions and fallback messaging (audio_utils already attempts static_ffmpeg).
- Data privacy: add a retention/consent dialog and option to purge uploads/reports.

## Deliverables
- Stable Streamlit demo with instructions to run locally.
- Test suite covering core audio & scoring functions.
- Exportable PDF report examples in `reports/`.
