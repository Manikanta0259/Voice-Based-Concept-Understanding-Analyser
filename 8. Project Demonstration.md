# 8. Project Demonstration

## Purpose
Step-by-step demonstration notes and example walkthroughs to present VBCUA to instructors, students or evaluators.

## Demo script (10–15 minutes)
1. Launch demo:
   - streamlit run source/app.py
2. Sign up / Log in
3. Create a reference concept:
   - Click "Create Custom Concept...", enter title and reference explanation + keywords
4. Upload audio:
   - Use `samples/` generated by `source/generate_samples.py` or an actual sample recording
   - Preview waveform
5. Run analysis:
   - Transcription (Whisper), semantic similarity, keyword coverage, fluency metrics
   - Show the "Final Evaluation" card and the waveform plot
6. Download PDF report:
   - Click "⬇️ Download PDF Report" and open it to show layout, metrics, and coaching suggestions
7. Show history:
   - Re-load previous runs from the sidebar and demonstrate reproducibility

## Example demo assets
- Synthetic good/weak audio created by `source/generate_samples.py`
- Example pre-generated reports in `reports/` to show different outcomes

## Presentation tips
- Emphasize how semantic similarity and keyword coverage complement each other (concept vs exact terms).
- Demonstrate coaching suggestions that come from scoring_engine and how they can be tuned.
- Discuss limitations: ASR accuracy, single-language focus, dataset biases.

## Next steps post-demo
