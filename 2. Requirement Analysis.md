# 2. Requirement Analysis

## Purpose
List functional and non-functional requirements for VBCUA and map them to modules in the repository.

## Functional requirements
- User management: sign up, login, session tracking → `source/database.py`, `source/app.py`
- Upload audio and preview waveform → `source/app.py`, `source/audio_utils.py`
- Speech-to-text transcription → `source/speech_to_text.py`
- Semantic similarity vs reference and keyword coverage → `source/semantic_eval.py`, `source/concepts.py`
- Fluency metrics and filler detection → `source/audio_utils.py`, `source/scoring_engine.py`
- Persist evaluation results and reports → `source/database.py`, `source/report_generator.py`
- Generate downloadable PDF report → `source/report_generator.py`

## Non-functional requirements
- Lightweight setup (Python + Streamlit, single-file DB)
- Reproducible test artifacts (synthetic WAV generation) → `source/generate_samples.py`
- Secure password storage (PBKDF2) → `source/database.py`
- Cross-platform audio support (ffmpeg + librosa) → `source/audio_utils.py`

## Priority & MVP scope
- MVP: upload audio → transcribe → semantic & keyword analysis → scoring → save + PDF.
- Post-MVP: role-based dashboards, batch processing, multi-language support.

## Open questions
- Which Whisper model(s) should be bundled or expected to run locally vs remote?
- How to version report template changes so historic reports remain comparable?
- Privacy / storage retention policy for uploads and reports.
