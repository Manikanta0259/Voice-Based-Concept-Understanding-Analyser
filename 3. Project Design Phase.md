# 3. Project Design Phase

## Purpose
Design notes, system architecture, data model highlights and the mapping of components to repo files.

## Architecture overview
- UI: Streamlit (`source/app.py`) — orchestrates flows: auth, upload, analysis, report download.
- Audio layer: `source/audio_utils.py` — loading, normalization, RMS/ZCR, pause detection, waveform plotting.
- ASR: `source/speech_to_text.py` — Whisper wrapper.
- NLP: `source/semantic_eval.py` + `source/concepts.py`
- Scoring & pedagogy: `source/scoring_engine.py`
- Persistence: SQLite & helper API in `source/database.py` (stores users, concepts, audio metadata, transcripts, metrics, evaluation results, reports)
- Reporting: `source/report_generator.py` (ReportLab renderer)
- Dev/test artifacts: `source/generate_samples.py`, `tests/` (placeholder)

## Data model highlights
- Users -> Sessions -> Audio files -> Transcripts -> Semantic similarities, audio features -> Evaluation results -> Reports
- Passwords stored with PBKDF2 (see `source/database.py`)

## Design decisions & rationale
- Single-file SQLite (`vbcua.db`) keeps deployment simple for demos.
- Modular design (audio/nlp/scoring/reporting) so each piece can be replaced or improved independently.
- PDF reports rendered server-side using ReportLab to provide a portable artifact for educators.

## Integration points
- `app.py` imports and orchestrates every module — use it as the canonical runtime example.
- `report_generator.py` reuses scoring logic to recompute sub-scores if needed.
