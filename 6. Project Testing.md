# 6. Project Testing

## Purpose
Testing plan, suggested test cases, and CI considerations for VBCUA.

## Testing scope
- Unit tests: audio utilities, concept encoding/decoding, scoring logic, DB CRUD helpers.
- Integration tests: end-to-end analysis of a synthetic audio file through the pipeline (load -> transcribe (mock) -> semantic_eval -> scoring -> report generation).
- UI tests (optional): interactive Streamlit tests using streamlit-testing helpers or manual acceptance tests.

## Suggested test files
- tests/test_audio_utils.py
  - test_load_and_preprocess_audio_normalizes_and_resamples
  - test_analyze_audio_features_detects_pauses_for_sample_patterns (use `source/generate_samples.py` outputs)

- tests/test_concepts.py
  - test_encode_decode_roundtrip

- tests/test_scoring_engine.py
  - test_detect_filler_words
  - test_calculate_scores_edge_cases

- tests/test_database.py
  - use a temporary DB file (or sqlite :memory:) to test register/authenticate/save_evaluation/get_evaluation_detail

- tests/test_report_generator.py
  - generate a report for a minimal `detail` dict and assert file exists and > 0 bytes

## CI recommendations
- Use GitHub Actions with a job that:
  - Sets up Python 3.10+ virtualenv
  - Installs requirements (`pip install -r source/requirements.txt`)
  - Runs pytest
  - Optionally caches venv or pip wheel cache

## Mocks & determinism
- Mock ASR (Whisper) calls in tests — do not call heavy models in CI.
- Use deterministic synthetic audio from `generate_samples.py` or pre-checked-in short WAV fixtures.
