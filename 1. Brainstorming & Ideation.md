# 1. Brainstorming & Ideation

## Purpose
Capture the original ideas, motivating problems, target users, and high-level features that drove the creation of VBCUA (Voice-Based Concept Understanding Analyser).

## Problem statement
Students and educators need an objective way to evaluate conceptual understanding delivered via spoken explanations. Existing assessments emphasize written responses and do not capture fluency, hesitation, or verbal coherence.

## Target users
- Students practicing verbal explanations and presentations
- Educators assessing oral understanding
- Researchers studying spoken-learning analytics
- Trainers / speech coaches

## Core ideas & feature candidates
- Speech-to-text transcription (OpenAI Whisper)
- Semantic similarity between transcript and reference (Sentence-BERT)
- Keyword coverage (required concept terms)
- Fluency & voice metrics (pause detection, RMS energy, ZCR, filler words)
- Per-evaluation PDF reports (ReportLab) with waveform visualizations
- User accounts, history, and reusable reference concepts
- Lightweight Streamlit front-end for fast experimentation

## Key constraints & assumptions
- Use open models where possible (SBERT / Whisper variants)
- Keep runtime simple (SQLite database, local files for uploads/reports)
- UX should be iterative: upload → analyze → view report → coach

## Early brainstorming decisions
- Use Streamlit for rapid UI prototyping and demos (see `source/app.py`)
- Store data in SQLite (`source/vbcua.db`) with database helpers in `source/database.py`
- Separate concerns: audio utilities (`source/audio_utils.py`), NLP (`source/semantic_eval.py`), scoring (`source/scoring_engine.py`) and reporting (`source/report_generator.py`)
